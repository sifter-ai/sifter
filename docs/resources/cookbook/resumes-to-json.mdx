# Resume parsing

Extract structured candidate data from PDF resumes. Works on any layout — chronological, skills-first, table-based, or infographic.

## Setup

```bash
pip install sifter-ai
```

## Schema and extraction

```python
from sifter import Sifter

s = Sifter(api_key="sk-...")

sift = s.create_sift(
    name="Candidates",
    instructions="""
    Extract:
    - full_name (string)
    - email (string or null)
    - phone (string or null)
    - location (string or null)
    - linkedin_url (string or null)
    - years_of_experience (integer or null)
    - skills (array of strings)
    - languages (array of {language: string, level: string})
    - experience (array of {
        company: string,
        role: string,
        start: string (YYYY-MM),
        end: string or null,
        current: bool,
        description: string or null
      })
    - education (array of {
        institution: string,
        degree: string,
        year: integer or null
      })
    - open_to_remote (bool or null)
    """,
)

folder = s.create_folder("Applications — Q2 2025")
folder.add_sift(sift)
folder.upload("./resumes/")

sift.wait()
print(f"Parsed {len(sift.records())} resumes")
```

## Query across candidates

```python
# Natural language
result = sift.query("Who has Python and more than 5 years of experience?")
print(result)

# Programmatic filter — iterate all matching records across pages
seniors = sift.find(filter={
    "extracted_data.skills": {"$in": ["Python"]},
    "extracted_data.years_of_experience": {"$gte": 5},
    "extracted_data.open_to_remote": True,
})
for r in seniors.items:
    print(r["extracted_data"]["full_name"], r["extracted_data"]["years_of_experience"])
```

## Export to CSV

```python
sift.export_csv("candidates.csv")
```

## Process new applications automatically

```python
# Every file dropped in the folder is parsed automatically
folder.upload("./new_batch/")
```
