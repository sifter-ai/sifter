# Folders

A **Folder** is a document container. Its key property: it can be linked to multiple sifts. Upload a document once — every linked sift processes it automatically.

## Use cases

- **Multi-extractor pipelines**: a folder of contracts linked to both a "Parties" sift and a "Clauses" sift
- **Shared document library**: multiple teams share the same folder; each team's sift extracts what they need
- **Inbox automation**: point a webhook or Zapier trigger at a folder; every incoming file is automatically processed

## Uploading documents

Supported formats: **PDF**, **DOCX**, **HTML**, **TXT**, **MD**, **CSV**, and images (**PNG**, **JPG**, **TIFF**, **WEBP**). Maximum 50 MB per file.

<Tabs>
  <Tab title="Python SDK">
```python
# Upload a directory
folder.upload("./contracts/q1/")

# Upload a single file
folder.upload("./invoice.pdf")

# Upload from bytes
with open("./invoice.pdf", "rb") as f:
    folder.upload(f.read(), filename="invoice.pdf")
```
  </Tab>
  <Tab title="TypeScript SDK">
```ts
// Single file (Node.js path)
await folder.upload("./invoice.pdf");

// From bytes
import { readFileSync } from "fs";
const bytes = readFileSync("./invoice.pdf");
await folder.upload(bytes, { filename: "invoice.pdf" });

// From a browser File object
await folder.upload(fileInput.files[0]);
```
  </Tab>
  <Tab title="REST API">
```bash
# Single file
curl -X POST https://api.sifter.ai/api/folders/{id}/documents \
  -H "X-API-Key: sk-..." \
  -F "files=@invoice.pdf"

# Multiple files
curl -X POST https://api.sifter.ai/api/folders/{id}/documents \
  -H "X-API-Key: sk-..." \
  -F "files=@invoice1.pdf" \
  -F "files=@invoice2.pdf"
```
  </Tab>
</Tabs>

## Document status per sift

Each `(document, sift)` pair has its own processing status. A document can be `done` for one sift and `processing` for another.

| Status | Meaning |
|--------|---------|
| `pending` | Queued, not yet started |
| `processing` | Worker currently running |
| `done` | Extraction complete |
| `error` | Extraction failed (retried up to 3×) |
