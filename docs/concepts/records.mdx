# Records

A **Record** is a structured result produced when a document is processed by a sift. A single document can produce multiple records — for example, a contract with several parties or an invoice with multiple line items grouped as separate entries. It contains:

| Field | Description |
|-------|-------------|
| `id` | Record identifier |
| `sift_id` | The sift that produced this record |
| `document_id` | The source document |
| `extracted_data` | JSON object with extracted fields |
| `document_type` | Inferred document type (e.g. `"invoice"`) |
| `confidence` | Extraction confidence score (0–1) |
| `created_at` | ISO timestamp |

## Fetching records

<Tabs>
  <Tab title="Python SDK">
```python
for record in sift.iter_records():
    print(record["extracted_data"])
    print(record["confidence"])
```
  </Tab>
  <Tab title="TypeScript SDK">
```ts
const records = await sift.records({ limit: 100 });
records.forEach((r) => {
  console.log(r.extracted_data);
  console.log(r.confidence);
});
```
  </Tab>
  <Tab title="REST API">
```bash
GET /api/sifts/{id}/records?limit=50&offset=0
```
Response:
```json
{
  "items": [...],
  "total": 142,
  "limit": 50,
  "offset": 0
}
```
  </Tab>
</Tabs>

## Schema

The `extracted_data` shape is defined by the sift's inferred schema. Fields missing in a document appear as `null`.

Example for an invoice sift:
```json
{
  "client_name": "Acme Corp",
  "invoice_date": "2024-03-15",
  "total_amount": 1500.00,
  "vat_number": "IT12345678901"
}
```

## Verifying extractions

Every record carries a **citations** map — one entry per extracted field with the exact source snippet and a confidence score. The record detail modal in the UI shows this inline: a confidence badge (High / Medium / Low / Unverified) and the source text snippet alongside each value.

```json
"citations": {
  "supplier": { "document_id": "doc_...", "source_text": "OpenAI Ireland Ltd", "page": 1, "confidence": 0.98 },
  "total":    { "document_id": "doc_...", "source_text": "Total amount: 1500 EUR", "page": 2, "confidence": 0.92 }
}
```

Badge levels:
- **High** — `confidence ≥ 0.85`
- **Medium** — `0.60 ≤ confidence < 0.85`
- **Low** — `confidence < 0.60` or `inferred: true` (fuzzy match)
- **Unverified** — no citation entry for the field

Records extracted before citations were added show an empty map. Use `POST /api/sifts/{id}/reindex` to backfill them. See [Citations](/concepts/citations) for the full data shape and pipeline details.
