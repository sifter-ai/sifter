# Sifts

A **Sift** is Sifter's core abstraction. It defines:

- A **name** and optional description
- **Instructions** — natural language description of what to extract
- A **schema** — automatically inferred from the first processed document
- A persistent **table of results** — one or more records per document (a single document can yield multiple records depending on what the instructions extract)

## Creating a sift

<Tabs>
  <Tab title="Python SDK">
```python
sift = s.create_sift(
    name="Invoices",
    instructions="Extract: client name, invoice date, line items, total amount, VAT number",
)
```
  </Tab>
  <Tab title="REST API">
```bash
POST /api/sifts
{
  "name": "Invoices",
  "instructions": "Extract: client name, invoice date, line items, total amount, VAT number"
}
```

## Writing instructions

Instructions are plain natural language — write what you want extracted as you would explain it to a person. Sifter infers the schema automatically from the first processed document.

```
Extract the client name, invoice date, line items with description and unit price, total amount, and VAT number.
```

```
For each party in the contract, extract their full name, role, and signing date.
```

No need to specify types, formats, or field names — Sifter figures those out. If the inferred schema doesn't match what you expected, edit the instructions and re-process.

## Status lifecycle

| Status | Meaning |
|--------|---------|
| `active` | Ready and accepting new documents |
| `indexing` | One or more documents are being processed |
| `paused` | Processing is suspended |
| `error` | One or more documents failed extraction |

## Schema inference

Sifter infers the JSON schema from the first processed document. The schema is stored on the sift and shown in the UI. Subsequent documents are extracted using the same schema.

If a field is missing in a document, its value is `null` in the record.

## Uploading documents

Documents can reach a sift in two ways:

1. **Direct upload** — via `POST /api/sifts/{id}/upload` or `sift.upload()`
2. **Via a folder** — link a folder to the sift; every new document in the folder triggers extraction automatically

See [Folders](/concepts/folders) for the multi-sift pipeline pattern.
