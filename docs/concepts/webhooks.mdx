# Webhooks

Webhooks let you react to extraction events in real time — trigger a downstream pipeline,
notify a Slack channel, or sync results to another system without polling.

## Events

| Event | When |
|-------|------|
| `sift.document.processed` | A document was extracted successfully |
| `sift.document.discarded` | A document was skipped by the filter instructions |
| `sift.completed` | All documents in a sift finished processing |
| `sift.error` | A document extraction failed |
| `folder.document.uploaded` | A new document was added to a folder |

## Wildcard patterns

Sifter supports `*` (single segment) and `**` (any segments):

| Pattern | Matches |
|---------|---------|
| `sift.*` | `sift.completed`, `sift.error` |
| `sift.document.*` | `sift.document.processed`, `sift.document.discarded` |
| `**` | All events |

## Registering a webhook

<Tabs>
  <Tab title="Python SDK">
```python
s.register_hook(
    events=["sift.document.processed", "sift.document.discarded", "sift.error"],
    url="https://example.com/hooks/sifter",
    sift_id="sift-abc",  # optional: scope to one sift
)
```
  </Tab>
  <Tab title="TypeScript SDK">
```ts
await s.registerHook(
  ["sift.document.processed", "sift.document.discarded", "sift.error"],
  "https://example.com/hooks/sifter",
  { siftId: "sift-abc" }, // optional: scope to one sift
);
```
  </Tab>
  <Tab title="REST API">
```bash
POST /api/webhooks
{
  "events": ["sift.document.processed", "sift.document.discarded", "sift.error"],
  "url": "https://example.com/hooks/sifter",
  "sift_id": "sift-abc"
}
```
  </Tab>
</Tabs>

## Payload

Every event arrives as a POST with this JSON body:

```json
{
  "event": "sift.document.processed",
  "payload": {
    "status": "processed",
    "document_id": "doc-xyz",
    "sift_id": "sift-abc",
    "record_count": 1,
    "records": [
      {
        "id": "rec-123",
        "document_type": "invoice",
        "confidence": 0.97,
        "fields": {
          "client_name": "Acme Corp",
          "invoice_date": "2024-03-15",
          "total_amount": 1500.00
        }
      }
    ]
  }
}
```

The `status` field is present on every document event: `"processed"`, `"discarded"`, or `"error"`.
For `sift.document.discarded` the payload includes a `reason` string instead of `records`.
For `sift.error` it includes an `error` string.

See [Webhooks Reference](/integrations/webhooks-reference) for all payload schemas and delivery details.
