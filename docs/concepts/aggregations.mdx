# Aggregations

**Aggregations** are saved, named queries that summarize your extracted records. They are backed by MongoDB aggregation pipelines generated from natural language.

## Creating an aggregation

In the UI: open a sift → **Aggregations** tab → type a question → **Save**.

Via API:
```bash
POST /api/aggregations
{
  "sift_id": "sift-abc",
  "name": "Revenue by client",
  "natural_language": "Total amount by client name, sorted descending"
}
```

Sifter translates the natural language into a MongoDB pipeline and stores it. The pipeline runs on demand against the current records.

## Running an aggregation

```bash
GET /api/aggregations/{id}/run
```

Returns the pipeline result as a JSON array.

## Use cases

- "Total by vendor per month"
- "Average confidence by document type"
- "Top 10 clients by invoice volume"
- "Documents with missing VAT number"

## vs. Query

[Queries](/concepts/queries-and-chat) are one-off natural language questions that return results immediately without saving. Aggregations are saved pipelines you run repeatedly as new documents arrive.
