# Webhooks Reference

## Events

| Event | Trigger |
|-------|---------|
| `sift.document.processed` | A document was extracted successfully |
| `sift.document.discarded` | A document was skipped by the filter instructions |
| `sift.completed` | All documents in a sift finished processing |
| `sift.error` | A document extraction failed |
| `folder.document.uploaded` | A new document was added to a folder |

## Wildcard patterns

| Pattern | Matches |
|---------|---------|
| `sift.*` | `sift.completed`, `sift.error` |
| `sift.document.*` | `sift.document.processed`, `sift.document.discarded` |
| `**` | All events |

## Payload schema

All events share this envelope:

```json
{
  "event": "sift.document.processed",
  "payload": { ... }
}
```

Every `payload` includes `status`, `document_id`, and `sift_id`.

### `sift.document.processed`

Fired when a document is extracted successfully. The `records` array contains
all records extracted from that document — typically one per document.

```json
{
  "event": "sift.document.processed",
  "payload": {
    "status": "processed",
    "document_id": "doc-xyz",
    "sift_id": "sift-abc",
    "record_count": 1,
    "records": [
      {
        "id": "rec-123",
        "document_type": "invoice",
        "confidence": 0.97,
        "fields": {
          "client_name": "Acme Corp",
          "invoice_date": "2024-03-15",
          "total_amount": 1500.00
        }
      }
    ]
  }
}
```

### `sift.document.discarded`

Fired when the sift's filter instructions determined the document does not
belong in this sift (e.g. a non-invoice uploaded to an invoice sift).

```json
{
  "event": "sift.document.discarded",
  "payload": {
    "status": "discarded",
    "document_id": "doc-xyz",
    "sift_id": "sift-abc",
    "reason": "Document is a bank statement, not an invoice."
  }
}
```

### `sift.completed`

```json
{
  "event": "sift.completed",
  "payload": {
    "sift_id": "sift-abc",
    "document_count": 42,
    "record_count": 42
  }
}
```

### `sift.error`

```json
{
  "event": "sift.error",
  "payload": {
    "status": "error",
    "document_id": "doc-xyz",
    "sift_id": "sift-abc",
    "error": "PDF could not be parsed: encrypted file"
  }
}
```

### `folder.document.uploaded`

```json
{
  "event": "folder.document.uploaded",
  "payload": {
    "document_id": "doc-xyz",
    "filename": "contract.pdf",
    "folder_id": "folder-abc"
  }
}
```

## Delivery

- Sifter delivers webhooks via HTTP POST to your URL
- Your endpoint must return a 2xx status within 10 seconds
- On failure: retried up to 3 times with exponential backoff (5s, 30s, 5min)
- After 3 failures: delivery is abandoned (no dead letter queue in v1)

## Registering

```bash
POST /api/webhooks
{
  "events": ["sift.document.processed", "sift.document.discarded", "sift.error"],
  "url": "https://example.com/hooks/sifter",
  "sift_id": "sift-abc"   # optional — omit to receive events from all sifts
}
```
