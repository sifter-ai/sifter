# Python SDK

The `sifter-ai` SDK is a pure HTTP client that connects to a running Sifter server (cloud or self-hosted).

## Install

```bash
pip install sifter-ai
```

## Client

```python
from sifter import Sifter

s = Sifter(
    api_url="https://sifter.run",      # or http://localhost:8000 for self-hosted
    api_key="sk-...",                   # or set SIFTER_API_KEY env var
)
```

## Error handling

The SDK exposes a unified surface across OSS and Cloud. Cloud-only methods (connectors, shares, billing, audit log) raise specific errors when the backend doesn't support them or when your plan doesn't include them.

```python
from sifter import Sifter
import httpx

s = Sifter(api_url="...", api_key="sk-...")

try:
    sift = s.get_sift("sift-id")
    results = sift.query("Total by client")
except httpx.HTTPStatusError as e:
    if e.response.status_code == 402:
        print("Upgrade required for this feature.")
    elif e.response.status_code in (401, 403):
        print("Check your API key.")
    else:
        raise
```

| Exception | When it fires |
|-----------|---------------|
| `NotAvailableError` | Called a Cloud-only method against an OSS backend (endpoint returns 404/501). |
| `PlanLimitError` | Cloud backend returned `402 Payment Required` — current plan doesn't include the feature or the call exceeds the plan quota. Has `required_plan` and `limit` attributes. |
| `APIError` | Generic 4xx/5xx with message from the server. |
| `AuthError` | 401/403 — check your API key. |

<Info>
Autocomplete surfaces every method on both OSS and Cloud. The runtime decides if the call succeeds — so you can write one codebase that targets either backend.
</Info>

## Sifts

```python
# Create
sift = s.create_sift("Invoices", "client, date, total amount, VAT number")
sift = s.create_sift("Invoices", "client, date, total amount, VAT number", description="AP invoices 2024")

# Read — one page (items are SiftHandle objects)
page = s.list_sifts(limit=50, offset=0)   # returns Page(items, total, limit, offset)
print(f"{page.total} sifts total")
for sift in page.items:
    print(sift.name)

# Update / delete
sift = s.get_sift("sift-id")
sift.update(name="Invoices 2024")
sift.delete()
```

## Documents and results

```python
sift.upload("./invoices/")                                    # file or directory
sift.upload(pdf_bytes, filename="invoice.pdf")                # raw bytes
sift.upload("./invoices/", on_conflict="fail")                # default: raise 409 if file already exists
sift.upload("./invoices/", on_conflict="replace")             # replace & re-extract if file already exists
sift.wait(poll_interval=2, timeout=300)  # block until done

# One page of records
page = sift.records(limit=100, offset=0)   # Page(items, total, limit, offset, next_cursor)

# Iterate all records across pages (recommended for large datasets)
for record in sift.iter_records():
    print(record["extracted_data"])

# Cursor-based filter query
page = sift.find(
    filter={"extracted_data.total": {"$gt": 1000}},
    sort=[("extracted_data.date", -1)],
    limit=50,
)

# Count records
n = sift.records_count()
n_filtered = sift.records_count(filter={"extracted_data.status": "unpaid"})

# Natural-language query (generates + runs a MongoDB aggregation pipeline)
results = sift.query("Total by client this month")

# Raw MongoDB aggregation
totals = sift.aggregate([
    {"$group": {"_id": "$extracted_data.client", "total": {"$sum": "$extracted_data.total"}}},
    {"$sort": {"total": -1}},
])

sift.export_csv("invoices.csv")
```

## Extraction control

Trigger and poll extraction for a single document:

```python
sift.extract("doc-xyz")

status = sift.extraction_status("doc-xyz")  # "queued" | "running" | "completed" | "failed"
```

## Citations

Every extracted field is anchored to a source text, page, and confidence score:

```python
record_handle = sift.record("rec-abc123")
citations = record_handle.citations()
# {"vendor": {"source_text": "Acme Corp", "page": 1, "confidence": 0.98}, ...}
```

See [Citations](/concepts/citations) for the full shape.

## Record corrections

Override extracted values and reset them to the original extraction when needed.

```python
record = sift.record("rec-abc123")

# Correct one or more fields — returns the updated record dict
updated = record.correct({"total": 1234.50, "currency": "EUR"})

# Reset one or more fields back to the original extracted value
updated = record.reset("total", "currency")
```

`correct()` stores corrections with scope `"local"` (only this record). The corrected values are reflected immediately in the record's `extracted_data` and are visible in the UI.

<Info>
Corrections are tracked with the identity of the caller (API key or user). The UI shows who corrected a field and when.
</Info>

## Folders

```python
folder = s.create_folder("/q1-contracts")
folder.add_sift(sift)
folder.upload("./contracts/q1/")         # directory
folder.upload(pdf_bytes, filename="nda.pdf")  # bytes

# Documents (one page — paginate with offset for more)
page = folder.documents(limit=50, offset=0)
for doc in page.items:
    print(doc["filename"])

folder.sifts()          # Page of linked sifts (items are SiftHandle objects)
folder.delete()
```

## Multi-sift pipeline

```python
parties = s.create_sift("Parties", "contracting parties, dates, signatories")
clauses = s.create_sift("Clauses", "non-compete, termination conditions")

folder = s.create_folder("/contracts-2024")
folder.add_sift(parties)
folder.add_sift(clauses)
folder.upload("./contracts/")

parties.wait()
print(parties.records())
```

## One-liner

Creates a temporary sift, uploads, waits, returns records, then **deletes the sift**. Use this for fire-and-forget jobs where you don't need the sift to persist.

```python
records = s.sift("./invoices/", "client, date, total")
```

If you need to query the data later (via the API, MCP, or NL query), use `create_sift()` instead so the sift persists.

## Event callbacks

Register local callbacks for events on a sift or folder. Callbacks fire during `wait()`.

```python
sift.on("sift.document.processed", lambda doc_id, record: print(record))
sift.on("sift.error", lambda doc_id, error: print(f"Error: {error}"))
sift.on("sift.*", lambda *args: print(args))  # wildcard

folder.on("folder.document.uploaded", lambda doc: print(f"New: {doc}"))
```

Wildcard patterns: `*` matches any single segment, `**` matches any number.

## Webhooks

Register server-side HTTP callbacks:

```python
s.register_hook(
    events=["sift.document.processed", "sift.error"],
    url="https://example.com/webhook",
    sift_id=sift.id,  # optional: scope to one sift
)
hooks = s.list_hooks()
s.delete_hook(hook_id)
```

See [Webhooks Reference](/integrations/webhooks-reference) for payload schemas.
