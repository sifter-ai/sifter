# MCP Server

Sifter exposes the [Model Context Protocol](https://modelcontextprotocol.io) so LLM clients can query your sifts, run aggregations, and fetch citations directly — no custom integration code.

Two connection modes:

| Mode | Transport | Endpoint | When to use |
|------|-----------|----------|-------------|
| **Remote (Cloud)** | Streamable HTTP | `https://sifter.run/mcp` | Sifter Cloud, Starter+ — zero install |
| **Local (self-host)** | stdio (`uvx sifter-mcp`) | your Sifter base URL | Self-hosted, or local dev |

<Info>
The remote endpoint is a **Starter+** Sifter Cloud feature. Free-plan API keys receive `402 Payment Required` on any tool call — upgrade at [/settings/billing](https://sifter.run/settings/billing).
</Info>

## Remote — Sifter Cloud

No installation. Point any MCP-aware client at the hosted endpoint.

**Endpoint:** `https://sifter.run/mcp`
**Transport:** Streamable HTTP (MCP standard)
**Auth:** OAuth 2.0 (recommended) or API key

### OAuth (recommended)

The Sifter MCP remote supports [OAuth 2.0](https://modelcontextprotocol.io/specification/2025-03-26/basic/authorization) as defined in the MCP spec. Clients that support OAuth can authenticate without an API key — the client handles the browser redirect and token exchange automatically.

**Claude Desktop / Claude Code**

Just add the URL — Claude handles the OAuth flow:

```json
{
  "mcpServers": {
    "sifter": {
      "type": "http",
      "url": "https://sifter.run/mcp"
    }
  }
}
```

On first use Claude will open a browser window, you log in to Sifter, grant access, and the token is stored automatically. No API key to copy.

**Claude Code (CLI)**

```bash
claude mcp add sifter --url https://sifter.run/mcp
```

**Cursor**

```json
{
  "mcpServers": {
    "sifter": {
      "type": "http",
      "url": "https://sifter.run/mcp"
    }
  }
}
```

### API key

If your MCP client doesn't support OAuth, use an API key. Generate one in **Settings → API Keys** — the same key unlocks REST, SDK, and MCP.

**Claude Desktop**

Add to `~/Library/Application Support/Claude/claude_desktop_config.json` (macOS) or `%APPDATA%\Claude\claude_desktop_config.json` (Windows):

```json
{
  "mcpServers": {
    "sifter": {
      "type": "http",
      "url": "https://sifter.run/mcp",
      "headers": {
        "Authorization": "Bearer sk-your-api-key"
      }
    }
  }
}
```

**Claude Code (CLI)**

```bash
claude mcp add sifter --url https://sifter.run/mcp \
  --header "Authorization: Bearer sk-your-api-key"
```

**Cursor**

```json
{
  "mcpServers": {
    "sifter": {
      "type": "http",
      "url": "https://sifter.run/mcp",
      "headers": {
        "Authorization": "Bearer sk-your-api-key"
      }
    }
  }
}
```

**ChatGPT / Gemini / other MCP clients**

Any client that speaks MCP over Streamable HTTP can connect. Point it at `https://sifter.run/mcp` — use OAuth if your client supports it, or add the `Authorization: Bearer` header with an API key.

## Local — self-hosted or dev

For self-hosted Sifter, run the MCP server as a local stdio process and point it at your own Sifter instance.

### Install

```bash
# Zero-install via uvx:
uvx sifter-mcp

# Or install globally:
pip install sifter-mcp
```

### Claude Desktop (local)

```json
{
  "mcpServers": {
    "sifter": {
      "command": "uvx",
      "args": ["sifter-mcp"],
      "env": {
        "SIFTER_BASE_URL": "http://localhost:8000",
        "SIFTER_API_KEY": "sk-your-api-key"
      }
    }
  }
}
```

### Cursor (local)

```json
{
  "mcpServers": {
    "sifter": {
      "command": "uvx",
      "args": ["sifter-mcp"],
      "env": {
        "SIFTER_BASE_URL": "http://localhost:8000",
        "SIFTER_API_KEY": "sk-your-api-key"
      }
    }
  }
}
```

### CLI options

```bash
uvx sifter-mcp --help

# Options:
#   --base-url    Sifter server URL (default: SIFTER_BASE_URL or http://localhost:8000)
#   --transport   stdio | sse | streamable-http (default: stdio)
#   --host        Host for HTTP transport (default: 0.0.0.0)
#   --port        Port for HTTP transport (default: 8001)
```

### Self-hosted HTTP endpoint

If you run your own Sifter instance with `sifter-mcp` installed, the Streamable HTTP endpoint is mounted at `/mcp` on the Sifter server. Authenticate with `Authorization: Bearer <your-api-key>`.

```
http://your-sifter-instance.com/mcp
```

## Available tools

All tools operate within the authenticated user's org.

| Tool | Description |
|------|-------------|
| `list_sifts` | List all sifts with name, instructions, and document/record counts |
| `get_sift` | Get sift metadata and inferred schema |
| `list_records` | Paginated records for a sift (max 100 per call) |
| `query_sift` | Natural-language query → records + citations |
| `aggregate_sift` | Run aggregation pipelines (sum, count, group by…) |
| `list_folders` | List all folders |
| `get_folder` | Folder metadata, linked sifts, document list |
| `get_document` | Document metadata + page list |
| `upload_document` | Upload a document to a folder (Starter+) |

## Resources

| URI | Description |
|-----|-------------|
| `sift://{id}/records` | All records for a sift as a JSON resource |

## Example prompts

Once connected, you can ask your LLM:

- *"List my sifts."*
- *"Show me the last 10 records from my Invoices sift."*
- *"What's the total revenue by client this month?"*
- *"Which folders have more than 50 documents?"*
- *"Find contracts signed by Acme that expire before March."*

The LLM picks the right tool (`list_sifts`, `query_sift`, `aggregate_sift`, etc.) and returns an answer grounded in your extracted records.
