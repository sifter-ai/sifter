# Sifter Cloud

Sifter Cloud is the managed version of Sifter at [sifter.run](https://sifter.run). The **OSS engine is the complete product** — Cloud doesn't add features, it adds *operations*: someone else runs the infrastructure, rotates the keys, and bills the LLM calls.

<Info>
Everything in the OSS engine — chat, spec-driven [dashboards](/concepts/dashboards), [webhooks](/concepts/webhooks), [SDK](/integrations/python-sdk), MCP stdio — is free forever under MIT. Cloud adds managed ops and a handful of team-oriented features on top.
</Info>

## What Cloud adds

<CardGroup cols={2}>
  <Card title="Remote MCP" icon="bolt" href="/integrations/mcp-server">
    Managed `https://sifter.run/mcp` endpoint. Any MCP-aware client (Claude, ChatGPT, Gemini, Cursor, Claude Code) connects with a single API key — no local install. **Starter+**.
  </Card>
  <Card title="Connectors" icon="plug" href="/concepts/connectors">
    Google Drive folder sync and per-org mail-to-upload addresses. Documents land in a Sifter folder the moment they arrive at the source.
  </Card>
  <Card title="Billing & usage" icon="credit-card" href="/cloud/billing">
    Stripe-backed plans, usage dashboards with 50/80/100% alerts, self-service upgrade/downgrade via the Stripe customer portal.
  </Card>
  <Card title="Mail-to-upload" icon="envelope" href="/cloud/mail-to-upload">
    Each folder gets `folder-xxx@in.sifter.run`. Forward an email, attachments become documents.
  </Card>
  <Card title="Shares" icon="share-nodes" href="/cloud/shares">
    Turn a chat answer, aggregation, or dashboard view into a shareable link — org-only, public, or password-protected. Optional expiry. PDF + email delivery.
  </Card>
  <Card title="Audit log" icon="clipboard-list" href="/cloud/audit-log">
    Every security-relevant action (billing, API key, sift, user, connector). Filterable by actor / action / time. **Pro+**.
  </Card>
  <Card title="SSO" icon="key" href="/cloud/sso">
    Google and GitHub single sign-on. One click from the login page. **Pro+**.
  </Card>
  <Card title="Managed hosting" icon="server">
    MongoDB, LLM credits, storage, TLS, backups — handled. Bring zero infrastructure.
  </Card>
</CardGroup>

## Cloud vs. OSS

| Capability | OSS | Cloud |
|-----------|-----|-------|
| Chat over your records | ✓ | ✓ |
| Spec-driven dashboards | ✓ | ✓ |
| Webhooks (HMAC + retry) | ✓ | ✓ |
| REST API + SDK (Python / TS) | ✓ | ✓ |
| MCP **stdio** (local) | ✓ | ✓ |
| MCP **remote** (`https://sifter.run/mcp`) | — | Starter+ |
| Google Drive connector | — | Starter+ |
| Mail-to-upload | — | Starter+ |
| Shares (link, email, PDF) | — | Starter+ |
| Stripe billing portal | — | ✓ |
| Google + GitHub SSO | — | Pro+ |
| Audit log | — | Pro+ |

See [Pricing](/overview/pricing) for full plan breakdown.

## When to use which

- **Self-host OSS** if you already run infra, want to pin LLM costs to your own keys, or need air-gapped / on-prem deployment.
- **Cloud** if you want the product without running MongoDB, rotating keys, tracking token spend, or wiring Stripe yourself — and you want the remote MCP endpoint and SSO for a team of more than one.
