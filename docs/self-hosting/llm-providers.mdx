# LLM Providers

Sifter uses [LiteLLM](https://docs.litellm.ai/) internally. Any LiteLLM-compatible provider works.

Each of the four agents (extractor, pipeline, chat, dashboard) can use a **different** model, API key, and base URL. If a task-specific value is not set, it falls back to `SIFTER_DEFAULT_*`.

## Provider quick reference

| Provider | Model string | Notes |
|----------|-------------|-------|
| OpenAI | `openai/gpt-4o` | Best all-round quality |
| Anthropic | `anthropic/claude-opus-4-7` | Excellent on complex documents |
| Google Gemini (AI Studio) | `gemini/gemini-2.5-flash` | Fast and cost-effective; set `GEMINI_API_KEY` |
| Vertex AI | `vertex_ai/gemini-2.5-flash` | Uses GCP service account / ADC; no API key needed |
| Azure OpenAI | `azure/gpt-4o` | Also set `AZURE_API_BASE` and `AZURE_API_VERSION` |
| Fireworks.ai | `openai/accounts/fireworks/models/...` | Set `SIFTER_DEFAULT_BASE_URL=https://api.fireworks.ai/inference/v1` |
| Ollama (local) | `ollama/llama3.2` | No API key; set `SIFTER_DEFAULT_BASE_URL=http://localhost:11434` |
| Any OpenAI-compatible | `openai/your-model` | Set `SIFTER_DEFAULT_BASE_URL` to the endpoint |

## Minimal single-provider setup

All four agents share the same model and key:

```bash
SIFTER_DEFAULT_MODEL=openai/gpt-4o
SIFTER_DEFAULT_API_KEY=sk-...
```

## Mixed-provider setup (recommended)

Use a vision-capable model for extraction and a faster model for the rest:

```bash
# Extraction — vision-capable, higher quality
SIFTER_EXTRACTOR_MODEL=gemini/gemini-2.5-flash
SIFTER_EXTRACTOR_API_KEY=AIza...

# Pipeline, chat, dashboard — fast and cheap
SIFTER_DEFAULT_MODEL=openai/gpt-4o-mini
SIFTER_DEFAULT_API_KEY=sk-...
```

## Google Gemini (AI Studio)

```bash
SIFTER_DEFAULT_MODEL=gemini/gemini-2.5-flash
GEMINI_API_KEY=AIza...
```

<Note>
`gemini/` prefix uses AI Studio credentials (`GEMINI_API_KEY`). For production GCP deployments prefer `vertex_ai/` with Application Default Credentials.
</Note>

## Vertex AI

No API key is needed — Sifter uses Application Default Credentials.

```bash
SIFTER_DEFAULT_MODEL=vertex_ai/gemini-2.5-flash
GOOGLE_APPLICATION_CREDENTIALS=/path/to/service-account.json
# VERTEXAI_PROJECT=my-gcp-project   # optional if set in ADC
# VERTEXAI_LOCATION=us-central1
```

## Azure OpenAI

```bash
SIFTER_DEFAULT_MODEL=azure/gpt-4o
SIFTER_DEFAULT_API_KEY=your-azure-key
AZURE_API_BASE=https://your-resource.openai.azure.com/
AZURE_API_VERSION=2024-08-01-preview
```

## Fireworks.ai (OpenAI-compatible)

```bash
SIFTER_DEFAULT_MODEL=openai/accounts/fireworks/models/llama-v3p3-70b-instruct
SIFTER_DEFAULT_API_KEY=fw_...
SIFTER_DEFAULT_BASE_URL=https://api.fireworks.ai/inference/v1
```

## Ollama (local)

```bash
SIFTER_DEFAULT_MODEL=ollama/llama3.2
SIFTER_DEFAULT_BASE_URL=http://localhost:11434
# No API key needed
```

<Note>
Local models work but may produce less consistent schemas than frontier models, especially for complex or multi-page documents.
</Note>
